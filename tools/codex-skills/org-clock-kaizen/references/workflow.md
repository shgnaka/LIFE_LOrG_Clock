# Workflow Reference

## Goal
Turn hardcoded values and design smells into implementation-ready tickets.

## Steps

1. Run scan script and inspect hits.
2. Classify findings using taxonomy.
3. Create ticket with required fields.
4. Validate Definition of Ready.
5. Regenerate indexes.

## Definition of Ready Checklist

- Evidence includes at least two `path:line` references.
- Acceptance criteria are objective and testable.
- Test plan includes at least one executable scenario.
- Out-of-scope is explicit.
