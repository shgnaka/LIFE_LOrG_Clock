ID: KZN-20260222-005
Title: Externalize user-facing UI and status strings into Android resources
Category: ui
Status: Done
Priority: P1
Impact: High
Effort: M
Updated: 2026-02-22

## Problem
User-facing strings are hardcoded across Compose screens and ViewModel status updates. This prevents consistent localization, complicates copy review, and increases the risk of mixed-language UX.

## Evidence
- app/src/main/java/com/example/orgclock/ui/state/OrgClockUiState.kt:58
- app/src/main/java/com/example/orgclock/ui/viewmodel/OrgClockViewModel.kt:257
- app/src/main/java/com/example/orgclock/ui/viewmodel/OrgClockViewModel.kt:391
- app/src/main/java/com/example/orgclock/ui/app/OrgClockScreen.kt:416
- app/src/main/java/com/example/orgclock/ui/app/OrgClockScreen.kt:871

## Hardcoded/Smell
- Literal UI labels are embedded in composables (`Text("...")` patterns).
- `UiStatus` messages are built from literal strings inside ViewModel business flow.
- Copy management is split between code and resources.

## Proposed Change
- Move all user-facing labels/messages to `app/src/main/res/values/strings.xml`.
- Introduce a message key/resource-id based status model for ViewModel (`UiStatus` should carry resource id + args, not final text).
- Keep format arguments (`%s`, `%d`) in resource entries and resolve at UI layer.

## Acceptance Criteria
- ViewModel no longer emits user-facing literal strings directly.
- Main UI screens no longer contain hardcoded user-visible copy except debug-only non-product text.
- All new/existing user-facing messages are defined in string resources with stable keys.
- Existing behavior and status semantics remain unchanged from user perspective.

## Test Plan
- Run `./gradlew :app:testDebugUnitTest` and update affected ViewModel tests to assert message keys/args.
- Run `./gradlew :app:connectedDebugAndroidTest` and update UI assertions to use resources.
- Manual smoke: verify key flows (root setup, start/stop/cancel, settings) still show expected messages.

## Out of Scope
- Adding multilingual translations beyond default resource set.
- Full UI copy rewrite.

## Dependencies
- Coordinate with KZN-20260222-002 if resource/message resolvers are injected via app graph.
