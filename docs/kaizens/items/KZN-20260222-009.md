Title: Introduce shared clock and zone providers for deterministic time behavior
Category: domain
Status: Done

Impact: Medium

Updated: 2026-02-22

## Problem
Production code calls `ZonedDateTime.now()` and `ZoneId.systemDefault()` directly in multiple layers (activity wiring, notification service, UI ticker). This causes time-dependent behavior to be scattered and harder to test deterministically.

## Evidence

- app/src/main/java/com/example/orgclock/MainActivity.kt:51
- app/src/main/java/com/example/orgclock/MainActivity.kt:60
- app/src/main/java/com/example/orgclock/notification/ClockInNotificationService.kt:123
- app/src/main/java/com/example/orgclock/notification/ClockInNotificationService.kt:238
- app/src/main/java/com/example/orgclock/ui/app/OrgClockScreen.kt:644
- app/src/main/java/com/example/orgclock/ui/app/OrgClockScreen.kt:648

## Hardcoded/Smell
- Direct system clock/zone calls are duplicated across components.
- Time dependencies are implicit instead of injectable.
- UI elapsed-time ticker and service elapsed-time logic are tightly bound to system time.

## Proposed Change
- Introduce a shared `ClockEnvironment` abstraction exposing:
  - `now(): ZonedDateTime`
  - `zoneId(): ZoneId`
- Provide a default system implementation and pass it through composition root.
- Refactor affected components to use the shared provider.

## Acceptance Criteria
- Direct `ZonedDateTime.now()` and `ZoneId.systemDefault()` calls are removed from listed app components (except the default environment implementation).
- Notification/service/UI flows use the shared provider for current time and zone decisions.
- Existing behavior remains unchanged with default system environment.
- Unit tests can inject a fixed environment to produce deterministic results.

## Test Plan
- Add/extend unit tests for notification elapsed-time and heading list refresh using fixed time.
- Run `./gradlew :app:testDebugUnitTest`.
- Manual smoke: verify start/stop/history and running panel elapsed times still update correctly.

## Out of Scope
- User-selectable timezone settings UI.
- Historical data migration between timezones.
