## Problem
Backup retention count and backup interval policy are hardcoded in repository defaults. This blocks environment-specific tuning and makes policy testing harder than necessary.

## Evidence
- app/src/main/java/com/example/orgclock/data/BackupPolicyConfig.kt:3
- app/src/main/java/com/example/orgclock/data/SafOrgRepository.kt:21
- app/src/main/java/com/example/orgclock/data/SafOrgRepository.kt:177
- app/src/main/java/com/example/orgclock/data/SafOrgRepository.kt:203
- app/src/main/java/com/example/orgclock/data/BackupPolicyDecisions.kt:3
- app/src/main/java/com/example/orgclock/data/BackupPolicyDecisions.kt:12
- app/src/test/java/com/example/orgclock/data/BackupPolicyDecisionsTest.kt:8

## Hardcoded/Smell
- `backupGenerations` default is fixed to `20`.
- Clock backup interval is fixed to `15 * 60 * 1000L`.
- Policy values are coupled to repository implementation details.

## Proposed Change
- Introduce `BackupPolicyConfig` (or equivalent) with:
  - `backupGenerations`
  - `clockBackupIntervalMs`
- Inject policy into `SafOrgRepository` with defaults preserving current behavior.
- Keep pruning and interval checks unchanged, only replace literal sources.

## Acceptance Criteria
- [x] Repository logic reads retention/interval from a policy object (not inline literals).
- [x] Default policy values remain equivalent to current behavior (`20`, `15 minutes`).
- [x] Tests can override policy values to exercise short interval and low retention branches.

## Test Plan
- Added/extended tests for:
  - pruning when generations is small
  - interval-based backup throttling
- Executed `./gradlew :app:testDebugUnitTest` (pass).

## Implementation Notes
- Introduced `BackupPolicyConfig(backupGenerations, clockBackupIntervalMs)` with defaults matching existing behavior.
- Replaced repository inline retention/interval literals with `backupPolicy` references.
- Extracted backup decision helpers into pure functions for deterministic unit testing.

## Out of Scope
- Changing backup file name format.
- Changing backup content/hash implementation.

## Dependencies
- None.
