ID: KZN-20260222-011
Title: Make notification scan resilient to per-file failures
Category: data
Status: Ready
Priority: P1
Impact: High
Effort: M
Updated: 2026-02-22

## Problem
Notification scan currently fails fast. If one `.org` file cannot be loaded, the entire scan result becomes failure and active clocks from other healthy files are hidden. This creates a high-risk blind spot for users who rely on notification visibility while working across multiple files.

## Evidence
- app/src/main/java/com/example/orgclock/notification/ClockInScanner.kt:26
- app/src/main/java/com/example/orgclock/notification/ClockInScanner.kt:44
- app/src/main/java/com/example/orgclock/notification/ClockInNotificationService.kt:131
- app/src/main/java/com/example/orgclock/notification/ClockInNotificationService.kt:133

## Hardcoded/Smell
- Scan loop couples per-file I/O errors to global failure (`getOrElse { return Result.failure(it) }`).
- Result type only represents all-success or all-failure and cannot express partial degradation.

## Proposed Change
- Add `ClockInScanResult` with `entries: List<ClockInEntry>` and `failedFiles: List<FileScanFailure>`.
- Change `ClockInScanner.scan` to continue scanning after per-file failure and aggregate failures.
- Keep scan as hard-failure only when initial file listing fails.
- Update `ClockInNotificationService` to show active entries even when `failedFiles` is non-empty, and append degraded-status context to logs/status summary.

## Acceptance Criteria
- [ ] When at least one file scan succeeds, scanner returns success with collected entries even if other files fail.
- [ ] Notification remains visible with successful active entries under partial scan failure.
- [ ] Per-file failures are observable through structured failure payload and service logging/status output.
- [ ] Existing behavior for full-scan success and full-scan failure remains unchanged.

## Test Plan
- Add unit tests in `ClockInScannerTest` for mixed success/failure files and verify both `entries` and `failedFiles`.
- Add/extend notification service tests for partial failure path to verify notification continues with active entries.
- Run `./gradlew :app:testDebugUnitTest`.

## Out of Scope
- Automatic repair/retry workflows for broken files.
- UI redesign of notification copy beyond degraded-status messaging.

## Dependencies
- None.
