## Problem
Notification polling interval and inbox display limits are embedded as literals in the foreground service. This makes runtime tuning and deterministic notification tests costly because values can only be changed by editing service code.

## Evidence
- app/src/main/java/com/example/orgclock/notification/NotificationServiceConfig.kt:3
- app/src/main/java/com/example/orgclock/notification/ClockInNotificationService.kt:60
- app/src/main/java/com/example/orgclock/notification/ClockInNotificationService.kt:98
- app/src/main/java/com/example/orgclock/notification/ClockInNotificationService.kt:199
- app/src/main/java/com/example/orgclock/MainActivity.kt:35
- app/src/main/java/com/example/orgclock/MainActivity.kt:92
- app/src/test/java/com/example/orgclock/notification/NotificationInboxLinesTest.kt:8

## Hardcoded/Smell
- Hardcoded behavior values: `SCAN_INTERVAL_MS = 5 * 60 * 1000L`, `MAX_LINES = 6`.
- The same class owns both orchestration logic and tuning policy.

## Proposed Change
- Introduce `NotificationServiceConfig` (or equivalent) with `scanIntervalMs` and `maxLines`.
- Inject config into `ClockInNotificationService` via composition root.
- Keep default values equivalent to current behavior and expose test overrides.

## Acceptance Criteria
- [x] `ClockInNotificationService` no longer defines `SCAN_INTERVAL_MS` or `MAX_LINES` literals.
- [x] Foreground loop delay and inbox truncation both read from injected config.
- [x] With default config values, notification behavior remains functionally identical.
- [x] At least one unit test verifies truncation behavior with `maxLines = 2`.

## Test Plan
- Added unit test for inbox truncation and `"...more"` behavior with `maxLines = 2`.
- Executed `./gradlew :app:testDebugUnitTest` (pass).
- Manual smoke: enable notification in app settings and confirm periodic refresh still occurs.

## Implementation Notes
- Implemented by introducing `NotificationServiceConfig(scanIntervalMs, maxLines)` with defaults matching existing behavior.
- `ClockInNotificationService.sync()` now accepts config and passes values through intent extras.
- Inbox line assembly was extracted into `buildInboxLines(...)` to make truncation logic deterministic and unit-testable.

## Out of Scope
- Redesigning notification copy or notification channel behavior.
