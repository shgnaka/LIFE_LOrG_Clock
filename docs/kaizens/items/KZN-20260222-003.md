## Problem
The App Distribution upload wait loop count and sleep cadence are hardcoded in workflow script, which hides timeout policy intent.

## Evidence
- .github/workflows/distribute-android.yml:163
- .github/workflows/distribute-android.yml:188

## Hardcoded/Smell
- `seq 1 60` and `sleep 2` are embedded literal values.
- Timeout policy is not centralized or documented next to knobs.

## Proposed Change
- Replace literals with env vars (`UPLOAD_POLL_MAX_ATTEMPTS`, `UPLOAD_POLL_SLEEP_SEC`) and defaults.
- Print resolved values in workflow logs.

## Acceptance Criteria
- Workflow behavior is identical under default values.
- Polling policy can be changed without touching loop body logic.
- Logs include effective poll config.

## Test Plan
- Run workflow in dry style by shellcheck/local script extraction if available.
- Validate YAML syntax and one manual dispatch run.

## Out of Scope
- Changing authentication architecture.

## Dependencies
- None.
