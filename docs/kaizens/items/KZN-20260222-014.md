Title: Replace message-based error tone detection with typed domain errors
Category: domain
Status: Done
Priority: P2
Impact: Medium
Effort: M
Updated: 2026-02-22

## Problem
Error tone mapping in ViewModel depends on message-string inspection (`contains("already")`). This is fragile: copy changes or localization updates can silently alter warning/error behavior. Domain failures should be represented by stable typed codes, not message text patterns.

## Evidence
- app/src/main/java/com/example/orgclock/ui/viewmodel/OrgClockViewModel.kt:404
- app/src/main/java/com/example/orgclock/domain/ClockService.kt:79
- app/src/main/java/com/example/orgclock/domain/ClockService.kt:94
- app/src/main/java/com/example/orgclock/domain/ClockService.kt:390

## Hardcoded/Smell
- Presentation semantics (warning vs error) are coupled to free-form exception messages.
- Domain layer does not provide stable machine-readable failure classification for clock operations.

## Proposed Change
- Introduce typed domain error contract (for example `ClockOperationCode` + `ClockOperationException`).
- Update `ClockService` failure branches to emit typed codes for expected cases (already running, invalid heading level, validation conflict, I/O failure).
- Update `OrgClockViewModel` to map status tone from typed code and remove message-substring checks.
- Preserve existing user-facing message resources and arguments.

## Acceptance Criteria
- [ ] `OrgClockViewModel` no longer uses message text matching to determine status tone.
- [ ] Already-running path is identified via typed code and mapped to `StatusTone.Warning`.
- [ ] Unexpected/unknown failures still map to `StatusTone.Error`.
- [ ] Existing status resource IDs and user-facing behavior remain functionally equivalent.

## Test Plan
- Add/extend `ClockServiceTest` to assert typed error codes for key failure branches.
- Add/extend `OrgClockViewModelTest` to verify tone mapping using typed exceptions.
- Run `./gradlew :app:testDebugUnitTest`.

## Out of Scope
- Full application-wide exception framework redesign.
- Rewriting all domain/service interfaces beyond clock operations.
